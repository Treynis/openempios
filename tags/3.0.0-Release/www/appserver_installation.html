JBoss Application Server Installation Instructions

OpenEMPI has only been tested on the JBoss Application Server although it is deployed as a J2EE application that can be hosted on any J2EE-compliant application server. OpenEMPI 1.0 exposes its interface using a few session beans that were implemented using the EJB 2.1 specification. OpenEMPI 2.0, if deployed as a J2EE application, utilizes the EJB 3.0 specification to expose its functionality using a few stateless session beans. If you need support on another J2EE application you may raise an issue in the Project Tracker or do the port and contribute instructions back to the project.

You can download the JBoss application server from the following link. We have only tested OpenEMPI against JBoss 4.0.5 and 4.2.3 although it may be supported on other versions as well. The instructions below assume that you have downloaded the 4.2.3 version of the application server which is currently available as jboss-4.2.3.GA.zip. Installation of the application server is as easy as expanding the zip file at an appropriate directory. In these instructions we assume the zip file is expanded under d:\servers\jboss-4.2.3.

This completes the installation of the application server software. Back to the installation instructions summary page.
